# Projects

Each startup lives in its own folder under `projects/{startup-slug}/`.

Minimum startup file set:

- `project.md`
- `problem.md`
- `icp.md`
- `solution.md`
- `validation.md`
- `strategy.md`
- `financials.md`
- `roadmap.md`
- `decisions.md`

Project-specific agent artifacts belong under:

- `projects/{startup-slug}/outputs/{AGENT_NAME}/`

Use `projects/_template/` as the base when spinning up a new startup.

Current active startup folders in this repo:

- `projects/signal/`
- `projects/creditbank/`
- `projects/startup-ops/`

Founders should normally enter through `MERIDIAN-ORCHESTRATOR`, which can:

- show a standup for the last open project
- continue that project or start a new one
- populate or refine project files
- launch recommended kickoff work and checkpoints
